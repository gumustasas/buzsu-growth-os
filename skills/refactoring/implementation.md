# Refactoring — BUZSU Uygulama Adımları

1. **Kapsam belgele** — tam olarak hangi dosya/fonksiyonlar değişecek, liste.
2. **Kapsam dışı iyileştirmeleri not et** — bunlar ayrı görev olarak kaydedilir.
3. **Davranış testi tasarla** — değişiklik sonrası aynı girdi → aynı çıktı nasıl doğrulanacak?
4. **Taslağı `drafts/code/` altına yaz** — branch + PR formatında.
5. Görev sonu raporu ekle.

## Onay Notu

Taslak: SAFE PATCH. Canlı uygulama: MAJOR.
