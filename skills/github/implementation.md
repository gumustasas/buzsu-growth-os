# GitHub — Branch, PR ve İnceleme Akışı — BUZSU Uygulama Adımları

## Adımlar

1. Kod/doküman değişikliği tamamlandığında git status/diff ile özet çıkar.
2. Kullanıcıya özet sun, onay sonrası commit oluştur (Co-Authored-By eklenmez, CLAUDE.md: model/oturum referansı commit'e yazılmaz).
3. Push yalnızca belirtilen feature branch'e yapılır; main'e asla.
4. PR yalnızca açık kullanıcı talebiyle açılır.

## Sınır / Onay Notu

Push ve PR açma her zaman kullanıcı onayı/talebi gerektirir; bu skill mevcut CLAUDE.md kurallarını GitHub iş akışı diline çevirir, yenisini eklemez.
